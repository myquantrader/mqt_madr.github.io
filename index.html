MULTI AVG DAILY RANGES

Know Your Daily Range — At a Glance, Draggable, Points or Ticks

MQT_AvgDailyRange displays the average daily range (High - Low) for up to four configurable periods in a simple on-chart panel. Choose to display values in Points or Ticks, toggle individual periods on/off, and drag the panel to the corner you prefer (with optional snap-to-corner). Built for NinjaTrader 8 with a lightweight, clean display that stays out of your way.

How It Works

1

Compute Daily Range

On each bar close, the indicator calculates the daily range as High - Low and adds it to rolling sums for each enabled period.

2

Maintain Rolling Averages

Each period keeps a rolling sum and bar count, trimming older contributions once the count exceeds the period length.

3

Format in Points or Ticks

Set DisplayUnits to Points (formatted with decimals) or Ticks (converted using TickSize and formatted as whole ticks).

4

Draw the Panel (Draggable)

The ADR panel is rendered using Draw.TextFixed. Optional mouse handlers let you drag it to the nearest corner, or lock it in place.

Frequently Asked Questions

What does “Average Daily Range” mean in this indicator?

ADR is calculated as the average of the prior daily ranges, where each daily range is High - Low. The indicator maintains separate rolling averages for up to four lookback periods.

Can I show ticks instead of points?

Yes. Set Display Units (DisplayUnits) to Ticks. Values will be converted using the instrument’s TickSize and displayed as whole ticks.

Can I hide some of the ADR lines?

Yes. Use ShowPeriod1 through ShowPeriod4 to show only the periods you want.

How do I move the ADR panel?

Click and drag the panel. If Snap To Corner is enabled, it will snap to the nearest corner while dragging. If the panel won’t move, check Lock Panel.

Does this plot lines on the chart?

No plots are required. The values are displayed in a fixed text panel using Draw.TextFixed, keeping your chart clean and readable.

Will it work on intraday charts?

The indicator uses the currently loaded bar series. For best “daily range” meaning, apply it to a Daily series, or ensure your data series context matches how you want to interpret “daily” range.
